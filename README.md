# CodeBridge

CodeBridge 是一个**去中心化的跨设备验证码与消息同步工具**，由 Android 手机端和 Windows 桌面端组成。它把手机收到的验证码短信、普通短信、App 通知，以及本机保存的 TOTP 动态验证码，安全地同步到你授权的设备节点。

核心理念：**不依赖任何云端服务器**，让多台手机、多台电脑之间安全、可控地共享临时验证码和 2FA 信息。设备之间通过局域网、可选的 Tailscale 地址和受信节点中继建立同步拓扑，所有业务消息仅在用户授权的节点之间流转，并全程加密。

> 当前版本：`1.0.30`

## 快速了解

| | |
|---|---|
| **同步内容** | 验证码短信 · 普通短信 · App 通知 · TOTP 动态码 · 剪贴板 |
| **传输方式** | 局域网直连 · Tailscale 地址 · 受信节点多跳中继 |
| **安全模型** | 端到端 AES-256-GCM 加密 · 无云端服务器 · 设备级授权管理 |
| **平台** | Android（Kotlin 原生）· Windows（Electron 桌面端） |

## 适用场景

- 电脑登录网站时，手机收到短信验证码后自动推送到电脑端并复制到剪贴板。
- 多台手机和多台电脑之间按设备节点精细管理推送范围。
- 手机熄屏后仍尽量完成验证码、短信、通知转发。
- 将 Google Authenticator / otpauth TOTP 种子导入本机，并同步到授权设备。
- 在任意一台设备上复制的文本，自动出现在其它授权设备的剪贴板里。
- 使用局域网发现和二维码配对快速加入新设备节点。
- 在没有公网服务器的情况下，通过局域网或 Tailscale 连接自己的设备。

## 核心功能

### 消息同步

- **验证码短信**：Android 端自动识别短信中的验证码，推送到目标设备。
- **全部短信**：用户可选择是否推送普通短信原文。
- **App 通知**：用户可选择是否通过 Android 通知监听服务推送 App 通知。
- **接收策略**：每个节点可独立选择接收验证码短信、普通短信、App 通知中的哪些内容。
- **来源标注**：桌面端会标注消息来源设备、来源应用/号码、目标节点和同步路径。

### TOTP 动态验证码

- 支持标准 `otpauth://totp` 二维码导入。
- 支持 Google Authenticator 迁移二维码批量导入。
- 支持剪贴板图片和上传图片解析二维码。
- 支持本地生成 TOTP，并在界面中展示倒计时。
- 支持多条 TOTP 置顶、编辑、删除和重复导入覆盖更新。
- 来源设备删除自己提供的 TOTP 后，可向其它节点同步删除状态。

### 剪贴板全网同步

- 剪贴板在网络中作为单值寄存器，采用 **LWW（last-writer-wins）** 策略：每次复制产生新版本，节点只应用更新于已应用版本的内容，自动吸收旧值、乱序副本和回环副本。
- 支持文本剪贴板自动同步。
- 支持图片剪贴板同步（首版仅同步较小 PNG）。
- 本地仅存内容哈希，不保留明文历史。
- 各项同步可在设置中按设备独立开关。

### 多设备与拓扑

- 手机和电脑都作为对等设备节点参与拓扑，不再把电脑作为唯一目标。
- 支持一台手机推送到多台电脑，也支持多台手机推送到同一电脑。
- 支持手机到手机、电脑到电脑、手机到电脑之间的节点配对。
- 新版网络层采用应用层软总线模型：业务消息统一封装为 topic/envelope，再由路由层选择直连或中继。
- 支持 LAN / Tailscale 可达节点之间优先 P2P 直连；不可达时回退到受信节点 relay。
- 支持**多跳中继**：直连不可达的节点可经由受信节点转发。
- 拓扑变更自动传播，离线节点重新上线后可补同步状态。

### 自动更新

- 桌面端基于 electron-updater，启动后静默检查、后台下载，确认后一键安装。
- Android 端内置 APK 更新检查与下载。
- 两端均可在界面中手动触发「检查更新」。

### 桌面端体验

- Electron 桌面端常驻系统托盘。
- 收到验证码短信时弹出右下角气泡，并自动复制验证码。
- 普通短信和 App 通知在消息列表中按类型展示，不会误复制空验证码。
- 支持显示验证码或原始短信内容。
- 支持授权设备管理：禁用、恢复、撤销。

### Android 端体验

- Kotlin Android 原生客户端。
- 支持扫码配对、局域网发现、推送目标管理。
- 支持熄屏转发所需的前台服务、唤醒锁和 Wi-Fi 锁。
- 支持短信接收、通知监听、节点收件服务。
- 支持本节点发送/接收内容策略设置。

## 安全与隐私

- 默认不经过任何云端服务器，消息只在用户设备之间流转。
- 配对二维码中包含设备地址和配对密钥，只有扫码后的设备可以建立受信连接。
- WebSocket 业务消息使用 **AES-256-GCM** 加密。
- 手机和桌面端都保存设备身份和授权列表，可单独禁用或撤销设备。
- TOTP 种子属于高敏感信息，只有来源设备和被授权节点应持有。
- 剪贴板同步本地只存内容哈希，不保留明文历史。
- Android 通知监听权限需要用户在系统设置中手动授权。

## 架构概览

```text
        Android Phone A                          Android Phone C
   SMS / Notification / TOTP                 SMS / Notification / TOTP
        Clipboard                                 Clipboard
            |                                          ^
            |  encrypted message (AES-256-GCM)         |
            v                                          |
   ┌──────────────────┐         relay         ┌──────────────────┐
   │ Windows Desktop B │ <-------------------> │ Windows Desktop D │
   └──────────────────┘    (multi-hop)        └──────────────────┘

   传输层：局域网直连 / Tailscale 地址 / 受信节点多跳中继
```

当前网络架构为 **SoftBus 控制面 + P2P 数据面**：

- SoftBus 控制面负责设备发现、可信入网、拓扑 LSDB、权限策略、topic 路由和离线补同步。
- P2P 数据面优先使用 LAN 或 Tailscale 地址直接发送 `/bus/message` 和 `/bus/file/{id}`。
- 旧版 `verify_code`、`codebridge_relay`、`topology_delta` 协议仍保留，作为旧节点兼容和直连失败时的兜底。
- 首版不提供公网 DHT、STUN/TURN 或 NAT hole punching；跨网段建议使用 Tailscale。

## 安装使用

### Windows 端

1. 从 [Release 页面](https://github.com/happyfox-dot/Msg2Computer/releases) 下载最新安装包。
2. 双击运行安装程序。
3. 启动后进入「配对」页面。
4. 确认 Windows 防火墙允许本应用在局域网监听。
5. 等待页面显示二维码。

### Android 端

1. 从 [Release 页面](https://github.com/happyfox-dot/Msg2Computer/releases) 下载最新 APK。
2. 授权短信、通知、相机、前台服务等权限。
3. 如需推送 App 通知，进入系统「通知使用权」页面授权 CodeBridge。
4. 扫描 Windows 端二维码，或在局域网发现设备列表中加入节点。
5. 在「消息同步策略」里选择本节点发送和接收哪些内容。
