### 🔒 Android TOTP 严重 Bug 修复

修复了 4 个严重的 TOTP 功能 Bug！

#### Bug 1: 本地 TOTP 生成参数错误 ✅
**问题**: 
- 导入 SHA256/SHA512 或 8位验证码时，手机本地生成错误
- 手机端和桌面端验证码不一致

**修复**:
- `TotpUtil.kt` 现在支持动态参数（SHA1/SHA256/SHA512, 6/7/8位, 30/60秒）
- 所有调用处使用正确的参数生成验证码

#### Bug 2: 无法删除 TOTP ✅
**问题**:
- 只能添加，无法删除
- 导入错误后只能清除应用数据

**修复**:
- 长按 TOTP 卡片弹出删除确认对话框
- 确认后可安全删除单个 TOTP

#### Bug 3: TOTP 密钥明文存储 ✅
**问题**:
- 使用普通 SharedPreferences 明文存储密钥
- 桌面端加密，手机端反而明文

**修复**:
- 改用 `EncryptedSharedPreferences` (AES256-GCM 加密)
- 使用 Android Keystore 保护主密钥
- 自动从旧格式迁移到加密存储

#### Bug 4: 存储丢失参数 ✅
**问题**:
- 只存储 `label|secret` 两个字段
- algorithm/digits/period 全部丢失

**修复**:
- 创建完整的 `TotpEntry` 数据类
- 使用 JSON 格式存储所有参数
- 支持从旧格式自动迁移

## 🔧 技术细节

### TOTP 参数支持
| 参数 | 支持值 | 默认值 |
|------|--------|--------|
| algorithm | SHA1, SHA256, SHA512 | SHA1 |
| digits | 6, 7, 8 | 6 |
| period | 30, 60 | 30 |

### 加密存储
- **算法**: AES256-GCM
- **密钥管理**: Android Keystore
- **Key 加密**: AES256-SIV
- **Value 加密**: AES256-GCM
