# TOTP 功能完整升级总结

## 📅 更新日期
2026-06-09

## 🎯 升级概述

将手机端 TOTP 添加功能从**单一入口**升级为**三种灵活方式**，大幅提升用户体验和使用场景覆盖。

- ❌ 用户体验不够友好

- ✅ **智能图片识别**：从相册自动识别二维码
- ✅ **清晰的用户界面**：图标化选项，一目了然

## 📋 三种添加方式详解

### 方式一：📷 扫描二维码

**适用场景**：配置新账号、快速添加

**操作流程**：
1. 点击"添加 TOTP"
2. 选择"📷 扫描二维码"
3. 对准二维码扫描
4. 自动识别并保存

**特点**：
- ✅ 最快速（2步完成）
- ✅ 最准确（实时识别）
- ✅ 针对性提示："扫描 TOTP 二维码（Google Authenticator 格式）"

---

### 方式二：🖼️ 从相册导入（新增）

**适用场景**：
- 批量迁移 TOTP
- 恢复之前保存的二维码截图
- 导入别人发送的二维码图片
- 离线配置

**操作流程**：
1. 点击"添加 TOTP"
2. 选择"🖼️ 从相册导入"
3. 从相册选择包含二维码的图片
4. 自动识别并保存

**特点**：
- ✅ 支持截图导入
- ✅ 可批量准备后逐个导入
- ✅ 智能识别二维码位置
- ✅ 完整解析 TOTP 参数（issuer、algorithm、digits、period）

**支持的图片格式**：
- PNG、JPG、JPEG 等常见格式
- 二维码需清晰可见

---

### 方式三：⌨️ 手动输入密钥（增强）

**适用场景**：
- 只有文本密钥时
- 无法扫描或导入图片时
- 网页上复制的密钥

**操作流程**：
1. 点击"添加 TOTP"
2. 选择"⌨️ 手动输入密钥"
3. 输入标签（如 Google、GitHub）
4. 输入 Base32 密钥（如 JBSWY3DPEHPK3PXP）
5. 点击"保存"

**增强功能**：
- ✅ **自动规范化**：自动转大写、去空格、去横杠
- ✅ **完整验证**：
  - 空值检查 → "请输入 TOTP 密钥"
  - 格式验证 → "密钥格式无效，请输入有效的 Base32 密钥..."
- ✅ **输入优化**：
  - 密钥输入框自动大写
  - 禁用自动建议（安全）
  - 标签输入框首字母自动大写
- ✅ **示例提示**：输入框显示示例密钥格式

## 🔧 技术实现

### 修改的文件列表

| 文件 | 修改内容 | 说明 |
|-----|---------|------|
| **AndroidManifest.xml** | 新增相册权限 | READ_MEDIA_IMAGES (Android 13+) |
| **MainActivity.kt** | 核心功能实现 | 三种添加方式 + 图片识别 + 验证 |
| **QRScannerActivity.kt** | 优化扫描提示 | 根据来源显示不同提示 |
| **strings.xml** | 新增字符串资源 | 13 个新字符串 |

### 新增核心方法

| 方法 | 功能 | 行数 |
|-----|------|-----|
| `handleImageFromGallery()` | 处理相册图片，识别二维码 | ~60 行 |
| `parseTotpUri()` | 解析 otpauth:// URI | ~50 行 |
| `saveTotpSecretWithDetails()` | 保存完整 TOTP 信息 | ~30 行 |
| `showManualTotpInputDialog()` | 手动输入对话框（带验证） | ~70 行 |

### 使用的技术

1. **图片选择**：`ActivityResultContracts.GetContent()`
2. **二维码识别**：ZXing 库（项目已包含）
3. **权限管理**：分级权限请求（Android 13+ 适配）
4. **输入验证**：`TotpUtil.validateSecret()`
5. **URI 解析**：`android.net.Uri`

## 📦 构建说明

### 环境要求
- **JDK**: 17（必须！）
- **Gradle**: 8.5+
- **Android SDK**: compileSdk 34, minSdk 23

### 依赖说明
**无需新增依赖**，使用现有的：
```kotlin
implementation("com.google.zxing:core:3.5.3")
implementation("com.journeyapps:zxing-android-embedded:4.3.0")
```

### 构建步骤
```bash
# 方式一：使用 Gradle Wrapper
cd android
gradlew.bat assembleRelease   # Windows
./gradlew assembleRelease      # Linux/Mac

# 方式二：使用 Android Studio
打开 android 目录 → Build → Build APK(s)
```

### 签名
```bash
# 1. 对齐
zipalign -p -f 4 app-release-unsigned.apk app-release-aligned.apk

# 2. 签名（使用项目 keystore）
apksigner sign --ks ../release/codebridge-release.jks \
  --ks-key-alias codebridge \
  --ks-pass pass:<密码> \
  --key-pass pass:<密码> \
  --out app-release-signed.apk \
  app-release-aligned.apk

# 3. 验证
apksigner verify --print-certs app-release-signed.apk
```
